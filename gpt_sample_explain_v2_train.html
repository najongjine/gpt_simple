Lv.2 초등학생 모드

2. 숫자로만 생각하는 바보 컴퓨터 (토큰과 임베딩) 🧮

그런데 문제가 있습니다. 컴퓨터는 "사과", "바나나" 같은 한글이나 영어를 전혀 모릅니다. 오직 숫자(0과 1)만 알죠.
그래서 우리는 인간의 단어를 번호판(숫자)으로 바꿔서 컴퓨터에게 줘야 합니다. 이걸 토큰화(Tokenization)라고 합니다.

사람의 말 (Text) ➡️ 컴퓨터가 보는 단어장 번호 (Tokens)
"Hello, I am a robot." ➡️ [15496, 11, 314, 716, 257, 10700]
graph LR A[문장
"Hello"] -->|사전 뒤지기| B[단어의 번호
15496] B -->|뜻을 좌표로 변경| C[임베딩
(0.2, -0.4, ... 768개)]

단어를 숫자로 바꾼 후, 그 숫자의 의미를 엄청나게 긴 좌표(768개)로 풀어 설명해줍니다. 이걸 세상 사람들은 임베딩(Embedding)이라고 부릅니다. (사과=[빨갛다:0.9, 과일이다:0.8, 딱딱하다:0.3...])

Lv.3 중/고등학생 모드

3. 문맥을 기가 막히게 파악하는 방법! "어텐션(Attention)" 📡

한국어는 동음이의어가 많죠. "배가 고파서 배를 타며 배를 먹었다"
여기서 '배'가 사람 몸인지, 타는 배인지 과일인지 컴퓨터가 어떻게 알까요?

바로 여기서 어텐션(Attention)이라는 마법이 등장합니다!
어텐션은 문장 속 단어들이 서로 "너 나랑 얼마나 관련 있니?" 하고 소개팅을 하는 과정입니다.

Self-Attention:
"먹었다"라는 단어는 앞에 있는 "배(과일)"를 엄청 강하게 쳐다봅니다.
"타며"라는 단어는 앞에 있는 "배(선박)"를 강하게 쳐다보죠!
이 시선 교환을 통해 단어의 '진짜 뜻'을 깨닫게 됩니다.

그리고 MultiHeadAttention은 이런 소개팅을 여러 개의 조(Head)로 나눠서 동시에 진행하는 겁니다! 하나는 문법을 보고, 하나는 감정을 보고, 하나는 맥락을 봅니다.

이번에는 파이썬 코드 몇 줄 떨어뜨려 놓고 끝내지 않습니다.

설명 기준 기호: B (Batch Size, 배치 크기), T (Time Step / Sequence Length, 문장 길이), C (Channels / Embedding Dimension, 임베딩 차원)

🔹 0. 하이퍼파라미터 (GPTConfig)

🧸 [야간 유치원 비유] 교실 규칙 정하기
야간 유치원에 반을 새로 엽니다. "우리 반 아이들은 한 번에 최대 512단어까지만 기억할 수 있어(block_size). 그리고 머릿속에 아는 단어 수는 10만 개야(vocab_size). 생각은 12번 깊게 할 거고(n_layer), 한 번에 12가지 방향(문법, 감정 등)으로 동시에 생각할 거야(n_head)."
🖥️ [컴퓨터 공학 매칭] 하이퍼파라미터 튜닝 (GPU 메모리 설계)
네트워크를 메모리에 올릴 때 텐서(Tensor)의 차원(Shape)을 고정하는 상수(Constant) 선언부입니다. 여기서 선언된 숫자들이 아래 나오는 모든 행렬 크기를 결정하며 시스템 연산량(VRAM)과 직결됩니다. 
class GPTConfig:
    block_size = 512      # T: RNN이나 Transformer가 역전파를 수행할 최대 문장 길이 제한 (시야각 한계)
    vocab_size = 100277   # V: 최종 출력층(Linear)에서 확률을 계산할 클래스 표본의 수 (Softmax 결과 사이즈)
    n_layer = 12        # L: 블록 단위 모듈을 스택처럼 쌓을 깊이. 깊을수록 복잡한 추론력 확보.
    n_head = 12         # 병렬 연산을 위한 텐서 분할 개수. (Multi-Head Attention용)
    n_embd = 768        # C: 하나의 단어를 기계가 고차원 공간에 뿌릴 때 사용할 좌표(실수 변수)의 개수 
    dropout = 0.1       # 학습 과정에서 10%의 노드를 끄고 학습시켜, 특정 데이터만 달달 외우는 방지 기법
    batch_size = 8        # B: 8개의 독립된 문장을 거대한 행렬 하나로 구겨 넣어서 GPU 병렬 연산 효율을 높임

🔹 1. Head (가장 작은 어텐션 단위: 나와 너의 관계성 파악)

🧸 [야간 유치원 비유] 나랑 짝꿍 찾기! (Q, K, V)
유치원생 단어들이 원 안에 모여 앉아있습니다. 서로 친해지기 위해 각자 역할을 나눕니다.
단어A: "난 '사과'야! 난 달콤하고 빨간 걸 뜻해." (내 정체성 명찰 = Key)
단어B: "난 '먹었다'야! 난 과일이나 음식 단어를 찾고 있어." (내가 찾는 대상 조건 = Query)
단어B가 조건을 외치면, 단어A의 '사과' 명찰과 찰떡같이 맞아떨어집니다! 호감도(가중치 점수)가 확 올라가죠.
서로 친해지면 단어A는 자기가 가진 "과일의 느낌(가치: Value)"을 떼어내서 단어B에게 줍니다.
단, 아직 말하지 않은 미래의 단어와는 눈이 마주치지 못하게 안대(tril Mask)를 씌워 반칙을 막습니다!
🖥️ [컴퓨터 공학 매칭] Self-Attention과 내적(Dot Product) 행렬 연산
Transformer 구조의 핵심입니다. 자연어는 단어 순서보단 문맥(Context) 조합이 필수적입니다. 각 단어 좌표를 가중치(W)에 통과시켜 Query, Key 행렬을 얻고 서로 내적 시킵니다.
벡터 공간 내에서 두 성분이 일치할수록 값이 커지며(유사성 도출), 이를 Softmax를 통해 총합이 1.0이 되는 확률(분배 가중치)로 만들고 그 비율만큼 Value 행렬을 곱합니다. 이것이 동적 메모리 조회(Addressing) 메커니즘입니다.
특히 "말은 과거에서 미래로 흐른다(Autoregressive)"는 물리 법칙을 지키기 위해 우상단 행렬을 -inf로 날리는 Masking 배열 처리가 필수적입니다. 
class Head(nn.Module):
    def __init__(self, head_size):
        ...
        # [초기화 1] 단어가 가진 원래 지식(n_embd)을 '내 정체성 명찰(Key)' 크기(head_size)로 압축해주는 마법의 렌즈입니다.
        # [Input] x: (배치 B, 길이 T, 기본단어크기 n_embd)  --->  [Output] 반환될 변환기: (B, T, head_size) 크기로 출력 예정
        self.key = nn.Linear(config.n_embd, head_size, bias=False)
        
        # [초기화 2] "내가 찾고 싶은 짝꿍(Query)"의 조건을 담아내는 마법의 렌즈입니다.
        # [Input] x: (배치 B, 길이 T, 기본단어크기 n_embd)  --->  [Output] 반환될 변환기: (B, T, head_size) 크기로 출력 예정
        self.query = nn.Linear(config.n_embd, head_size, bias=False)
        
        # [초기화 3] 친해진 상대에게 넘겨줄 "나의 알짜배기 정보(Value)"를 만드는 마법의 렌즈입니다.
        # [Input] x: (배치 B, 길이 T, 기본단어크기 n_embd)  --->  [Output] 반환될 변환기: (B, T, head_size) 크기로 출력 예정
        self.value = nn.Linear(config.n_embd, head_size, bias=False)
        
        # [초기화 4] 미래 컨닝 방지용 '가림막(Mask)'을 만듭니다. 왼쪽 아래만 1이고, 오른쪽 위(미래)는 0인 직각삼각형 모양이에요.
        # [Input] block_size (예: 512)  --->  [Output] tril 배열: (block_size, block_size) 2차원 정사각형
        self.register_buffer('tril', torch.tril(torch.ones(config.block_size, config.block_size)))

    def forward(self, x):
        # [텐서 차원의 이해 (어디서 왔고 무엇을 의미하는가?)]
        # B (Batch Size, 모둠 수): 한 번에 훈련/처리할 "문장 세트의 개수" (보통 config.batch_size = 8)
        # T (Time/Sequence Length, 타임아웃): 하나의 문장 안에 들어있는 "단어 토큰의 총 개수" (보통 config.block_size = 512)
        # C (Channel/Embedding, 지식의 깊이): 각 단어 하나가 품고 있는 의미 "정보 벡터의 원소 개수" (보통 config.n_embd = 768)
        B, T, C = x.shape  # [Input] x: (B, T, C)
        
        # 1. 🔍 [질문과 답변 세팅] 입력 x를 각각 Query(찾는 조건)와 Key(내 명찰)로 변환합니다.
        k = self.key(x)     # [Input] x: (B, T, C)  --->  [Output] k: (B, T, head_size)
        q = self.query(x)   # [Input] x: (B, T, C)  --->  [Output] q: (B, T, head_size)
        
        # 2. 💘 [호감도(Score) 계산] "내가 찾는 조건(Q)" 이랑 "남들의 명찰(K)"을 매칭(행렬곱 `@`)시켜 점수를 매깁니다.
        # 계산 원리: Q (B, T, head_size) 곱하기 K를 뒤집은 행렬 (B, head_size, T)
        # (머리가 커질수록 점수가 폭주해서 Softmax가 망가지는 걸 막기 위해 차원수(head_size)의 루트값 C**-0.5 로 나눠 진정시킵니다)
        wei = q @ k.transpose(-2, -1) * (C**-0.5) 
        # [Output] wei: (B, T, T)  <-- 단어 T개가 서로를 얼마나 좋아하는지 적힌 T x T 크기의 점수판 완성이요!
        
        # 3. 🙈 [미래 컨닝 막기] 뒤에 나올 단어(아직 모름)와 미리 썸타면 반칙이죠? 가림막(tril)에서 0인 부분을 끝없는 절망(-inf)으로 덮습니다.
        wei = wei.masked_fill(self.tril[:T, :T] == 0, float('-inf'))
        # [Input] wei: (B, T, T)  --->  [Output] 덮어씌워진 wei: (B, T, T) (우상단 쪽이 모두 -inf로 덮임)
        
        # 4. 🧮 [점수를 100% 비율로 깔끔하게] -inf는 0%로 소멸시키고, 나머지 점수들은 합쳐서 100%(1.0)가 되도록 예쁘게 확률로 쪼갭니다.
        wei = F.softmax(wei, dim=-1)
        # [Input] 쌩 점수판 wei: (B, T, T)  --->  [Output] 정규화된 wei: (B, T, T) (각 줄을 다 더하면 무조건 1.0이 되는 최적의 조합 비율!)
        
        # 5. 🎁 [정보 조공 타임] 비율(wei)대로 짝꿍의 আসল 알짜배기 정보(Value)를 흡수합니다.
        v = self.value(x)   # [Input] x: (B, T, C)  ---> [Output] v: (B, T, head_size) (넘겨줄 예쁜 포장 상자)
        
        # (가중치 비율판) (B, T, T) 행렬 곱하기 (핵심 정보) (B, T, head_size)
        out = wei @ v       
        # [Output] out: (B, T, head_size)  <-- 다른 단어들의 문맥 정보와 "합체 완료"된 진화형 문맥 벡터 도출!
        
        return out

🔹 2. MultiHeadAttention (Attention 모듈의 병렬 집합)

🧸 [야간 유치원 비유] 12개의 방울 달린 안경
단어들이 짝꿍을 찾을 때 "문법" 기준으로만 짝을 찾으면 정보가 빈약하겠죠? 그래서 유치원생에게 12가지 색안경(Head)을 씌웁니다.
어떤 안경을 쓰면 "감정"에 반응하고, 다른 안경은 "주어-동사"에만 반응합니다. 12개의 각기 다른 기준으로 각자 친구를 찾은 다음, 맨 마지막에 이어 붙여서(cat) 원장의 종합 보고서로 통째로 제출합니다.
🖥️ [컴퓨터 공학 매칭] 파라미터 분산과 다중 피처 앙상블 조합 (Ensemble)
거대한 Attention 계산을 통째로 수행하면 가중치 표현력이 하나의 특징 공간으로 제한됩니다. 따라서 원래 차원을 `n_head` 개수로 나눈 뒤(예: 768차원을 64차원 벡터 12개로 분리), 독자적인 W행렬에서 각각 병렬 도출(연산 최적화)시켜 다채로운 특징 표상(Feature Representation)을 동시에 잡아냅니다. 그 뒤 메모리에선 단순히 마지막 차원을 연장(`torch.cat`)시켜 사이즈를 원래 차원(C)으로 복구시킵니다. 
class MultiHeadAttention(nn.Module):
    def __init__(self, num_heads, head_size):
        ...
        # Iterator를 사용해 작은 Head(64 크기) 모듈을 num_heads(12번) 만큼 생성 후 리스트에 주입.
        self.heads = nn.ModuleList([Head(head_size) for _ in range(num_heads)])
        # 각 헤드가 내놓은 결과를 찢어진 종이 붙이듯 하나로 섞어주는 선형 투영(Projection) 레이어.
        self.proj = nn.Linear(config.n_embd, config.n_embd)

    def forward(self, x):
        # 12개의 Heads로부터 나온 [B, T, 64] 크기의 텐서 12개를
        # 마지막 3번째 차원(dim=-1) 방향으로 다닥다닥 이어 붙임(cat).
        # 64 짜리가 가로로 12번 이어붙으니 64*12 = 768! -> 처음 입력때와 똑같은 차원 [B, T, 768] 원상복구.
        out = torch.cat([h(x) for h in self.heads], dim=-1)
        
        # 이어 붙이기만 해서는 정보가 단절되어 있어 Linear (행렬 [C, C] 곱)를 거쳐 텐서 축을 하나로 믹싱함
        out = self.proj(out)
        return out

🔹 3. FeedForward (깊이 곱씹어 생각하기)

🧸 [야간 유치원 비유] 나만의 텐트 속 명상 시간
강당에서 12개의 안경을 쓰고 친구들과 소통을 끝마쳤다면, 이번엔 각자 자기 방(텐트)으로 돌아갑니다. 다른 아이들과의 대화는 끊고, 오로지 앞서 얻은 힌트들을 내 머릿속에서 혼자 크게 4배 부풀려서 깊게 생각(상상)합니다. 그중 쓸모없는 잡생각은 망각해 버리고(ReLU), 알짜배기 깨달음만 챙겨서 다시 방 밖으로 나옵니다.
🖥️ [컴퓨터 공학 매칭] 비선형 다층 퍼셉트론 공간 팽창 (Non-linear MLP Expansion)
MHA 계층(앞의 과정)이 시간 통신망(Communication Across Tokens)이라면, FFN(현재 과정)은 토큰 혼자 특성 벡터 영역을 재가공하는 단일 노드 연산망(Computation Per Token) 역할입니다. 차원을 일시적으로 보통 4배 증폭(768 -> 3072)시켜 비정형 데이터 매핑 수용력(Capacity)을 극대화시킨 후, 0 이하 차원 값을 날려버리는 ReLU 활성 함수 스위치를 먹여 네트워크에 비선형성(Non-linear) 구조를 강제로 부여합니다. (이게 없다면 인공신경망 전체가 단순한 1차 방정식 그래프 모음집이 되어버려 복잡한 언어를 단 1%도 이해하지 못합니다.) 
class FeedForward(nn.Module):
    def __init__(self, n_embd):
        ...
        self.net = nn.Sequential(
            # 1. 공간 팽창 연산: 입력 C를 4배(4*C) 로 확장시킴으로써 추론 가능한 파라미터 바다 생성 [B, T, 768] -> [B, T, 3072]
            nn.Linear(n_embd, 4 * n_embd),
            
            # 2. 비선형성 활성 함수: 곡선 논리를 만들어 줌. (배열 값 중 0 이하는 강제로 0으로 파괴)
            nn.ReLU(),
            
            # 3. 투영/압축. 팽창된 공간에서 결론 도출이 끝난 핵심만 원래 C차원 생태계 규격으로 밀어넣음 [B, T, 3072] -> [B, T, 768]
            nn.Linear(4 * n_embd, n_embd),
        )

    def forward(self, x):
        return self.net(x)

🔹 4. Block (거대한 기초 샌드위치 조립)

🧸 [야간 유치원 비유] 1층 샌드위치 완성! 그리고 우주최강 단기기억법
유치원 일과는 [친구 파악하기(MHA)] -> [혼자 명상하기(FFWD)] 순서입니다. 그런데 이게 12번이나 반복되니까 아이들이 "내가 처음에 뭔 단어였지?" 까먹는 치매 현상이 옵니다. 그래서 "네 원래 정체는 이거였어!" 하고 옆으로 과거 모습을 슬쩍 컨닝 페이퍼마냥 전달해 줍니다. (기존의 나 + 새롭게 배운 나 = 잔차 연결) 이 마법 통로 덕분에 천재 유치원생이 만들어집니다.
🖥️ [컴퓨터 공학 매칭] Residual Connection과 Layer Normalization
딥러닝 최고의 발명품, 기울기 소실(Vanishing Gradient) 억제 장치입니다. 역전파(Backpropagation) 시 깊은 층으로 미분값을 계속 곱해 내리다 보면 에러값이 0.000...으로 수렴해 학습이 정지되어 버리는데, 수식 x = x + F(x) 형태의 잔차 연결법(Residual Method)은 훼손되지 않은 원본 $x$를 그대로 미분 패스로 뚫어(미분치 +1 추가) 아무리 깊은 층이라도 초기 정보를 상실하지 않게 보호하는 고속도로입니다.
추가로, 숫자 스케일이 제각각 커지며 치솟는 발산 현상을 방지하기 위해 각 토큰 벡터 요소를 평균 0, 분산 1 정규분포 구역 안으로 가둬 패킹하는 LayerNorm을 안전핀 목적으로 진입 직전마다 적용합니다(Pre-Norm 구조). 
class Block(nn.Module):
    def __init__(self, n_embd, n_head):
        ...
        self.sa = MultiHeadAttention(n_head, head_size)  
        self.ffwd = FeedForward(n_embd)                  
        # LayerNorm: 정규화. 각 토큰 축소 공간(C차원)마다 평균=0, 분산=1의 범위로 분포를 안정화 시킴
        self.ln1 = nn.LayerNorm(n_embd)                  
        self.ln2 = nn.LayerNorm(n_embd)

    def forward(self, x):
        # [Residual Connection 매직!] "x = x + ..."
        # 그 자리에 감싸진 sa망엔 정규화(ln)를 씌운 값만 통과시켜 변화량 F(x)값만 얻고 원본 본체를 보존.
        x = x + self.sa(self.ln1(x))
        # Feedforward 단계 역시 원본을 기억하도록 잔차 구조 덧셈을 유지함.
        x = x + self.ffwd(self.ln2(x))
        return x

🔹 5. GPT 종합 사령부 (전체 동작망 총괄)

🧸 [야간 유치원 비유] 유치원 풀 코스 학기 수료식
오늘 막 들어온 신입생은 그저 "나는[123]", "사과[456]" 같은 책 카탈로그 번호표에 불과합니다. 원장님이 이 단순한 번호표들에 '풍부한 의미 좌표(Token Emb)'와 '자리 순서표(Pos Emb)'를 매달아 교실로 투입합니다.
앞서 만든 거대한 Block 교실을 12번이나 통과하면 이 단어들은 "문장의 흐름과 세상 이치를 전부 깨우친 철학자 단어들"로 변신해 졸업합니다. 졸업 문에서 10만 단어 카탈로그를 펼친 뒤 "그래, 나 다음에는 누가 서는게 맞을까?" 투표판(Logits) 결과 점수를 내보고, 선생님이 준 정답지(Targets)와 비교해 "다 틀렸잖아!" 하고 벌점(Loss)을 매겨 다음날 더 똑똑하게 훈련되게 시스템을 굴립니다.
🖥️ [컴퓨터 공학 매칭] One-hot to Dense Vector화 변환, 그리고 Logits 분포 비용(Loss) 산출
NLP 토큰들은 단순한 범주형 정수 Index 배열 데이터일 뿐 의미 연산이 불가능합니다. 배열 [B, T]을 `nn.Embedding` 룩업 테이블을 거쳐 의미 깊이를 지닌 실수 공간 좌표인 [B, T, C](Dense Space Vector)로 스케일 업 해줍니다.
Transformer 구조망은 RNN이나 CNN처럼 태생적 시계열 파악 장치가 없으므로 강제로 거리 함수 배열인 `Positional Embedding` 벡터를 더해(Add) 모델이 인덱스 위치를 인식하도록 하드코딩 합니다.
최후방 출력 연결망(lm_head Linear)은 완성품 768 차원 벡터를 단어 전체 10만 개 클래스 도메인 구역으로 발산 매핑시켜 비통계형 확률 가중치인 로짓(Logit) 분포 구역을 형성합니다. 이를 실제 정답 레이블과 대차 대조하여 다항분류 체계의 베스트 프랙티스인 CrossEntropyLoss 목적함수 공식으로 비용 스칼라 오차값(`loss`)을 산출해냅니다. 
class GPT(nn.Module):
    def __init__(self):
        ...
        # 1. 의미 임베딩 테이블 (행렬: V x C) -> 들어온 인덱스 숫자에 맞춰 [B, T] -> [B, T, C]로 확장
        self.token_embedding_table = nn.Embedding(config.vocab_size, config.n_embd)
        # 2. 위치 임베딩 테이블 (행렬: T_max x C) -> 단순 배열인 텐서에 0~T번째라는 순서 개념을 더함
        self.position_embedding_table = nn.Embedding(config.block_size, config.n_embd)
        
        # 3. Block 12개의 직렬 연결 사슬 구조 생성
        self.blocks = nn.Sequential(*[Block(config.n_embd, config.n_head) for _ in range(config.n_layer)])
        
        # 4. 언어 모델링 출력층 변환 (Linear 파라미터 W: [C, V]) -> [B, T, C]차원을 [B, T, V]차원으로 변환
        self.lm_head = nn.Linear(config.n_embd, config.vocab_size)

    def forward(self, idx, targets=None):
        ...
        # [B, T] 원시 정수 배열을 넣으면 -> Vector 좌표계 [B, T, C]로 뻥튀기 확장.
        tok_emb = self.token_embedding_table(idx)  
        # 0, 1, 2... T-1 까지 만들어진 연속 배열 상수 텐서를 넣으면 포지셔널 좌표계 [T, C]로 뻥튀기 확장.
        pos_emb = self.position_embedding_table(torch.arange(T, device=config.device))  
        
        # 두 좌표계를 브로드캐스팅(Broadcasting, 자동 치환) 덧셈. [의미 벡터 + 위치 벡터] 융합 초기 노드 완성.
        x = tok_emb + pos_emb 
        # 12층 딥러닝 고속도로 연산 돌입. 잔차 연결 덕분에 [B, T, C] 차원은 소실되지 않음.
        x = self.blocks(x)       
        # 출력 차원 변환: 768짜리 내부 거대 특징 값들을 100277개의 실제 언어사전 단어 투표권 점수판 [B, T, Vocab_size] 로 스케일 역 변환 (Logits).
        logits = self.lm_head(x) 

        # -----[ 오차 최적화(Loss Validation) 훈련 모듈부 ]-----
        if targets is not None:
            # PyTorch 파이프라인 내부 `F.cross_entropy` API 연산 규약상, 배열 배치가 1차원이어야 하므로 조작합니다. 
            # 일자로 쭉 펴는 Flattening 진행: 배치 B와 문장 길이 T를 곱해서 하나로 나열합니다. -> 차원 [B*T, V_size] 
            logits = logits.view(B*T, config.vocab_size)      
            targets = targets.view(B*T) 
            
            # 정답 자리 인덱스가 가진 로짓 확률을 얼마로 파악했는지 역추적하고, 마이너스 로그 우도를 이용해 거리 격차 스칼라 값(Loss)을 반환.
            loss = F.cross_entropy(logits, targets)

        return logits, loss

🔹 6. 텍스트 추론기 (Auto-Regressive 생성 엔진)

🧸 [야간 유치원 비유] 뒷말 이어 짓기 복불복 게임
이미 선생님 역할을 끝낸 모델에게 "옛날 옛적에 호랑이가" 까지만 알려주고 단어 하나만 이어서 내뱉어보라고 시킵니다. AI가 10만 단어 점수판을 쓱 구경한 뒤 확률 상 제일 합리적이면서도 나이브 주사위 복불복(무작위 섞기)을 약간 가미해서 "담배를" 이란 단어를 로또 뽑듯 추출합니다! 그럼 기존 문장에 풀로 딱 붙여 "옛날 옛적에 호랑이가 담배를" 이라는 새 기본 문장을 만들죠. 이걸 지정한 턴 횟수만큼 끝도 없이 무한 도돌이표로 돌면 그럴싸한 소설 문서가 와르르 창작되는 겁니다!
🖥️ [컴퓨터 공학 매칭] 로짓 기반 범주 샘플링과 자기 회귀(Auto-Regressive) 스택 누적 연산
Backpropagation 기능이 잠기는 순수 평가/추론(Inference) 단계 루프입니다. 앞서 들어가는 Prompt(Context)가 너무 늘어나면 모델의 수학적 메모리 용량을 벗어나므로 `block_size`까지만 배열의 꼬리를 도려내는 `Slicing Truncation` 슬라이싱 기능이 필수입니다.
다음 단어를 맞추는 엔진은 과거 값들이 어떻게 됐느냐(단어의 궤적) 모델 구조망이 알 바가 아닙니다. 오직 가장 마지막 열의(인덱스 기준 `-1`) 분포 단 1개만 참조합니다.
만약 가장 점수 높은 1등만 채택하는 그리디 서치(argmax) 알고리즘을 사용하면 단어가 기계적으로만 무한 반복되는 디제너레이션(Degeneration)이 터지므로, 수학의 Softmax -> 확률 편차 조절 -> Multinomial(다항 카테고리 분포) 하 랜덤 샘플링 확률론을 통해 생성 문자열에 생동감과 창의성을 주입합니다. 이후 생성해낸 신규 1차원 숫자 배열을 원본 리스트 차원 축(dim=1) 기준으로 `cat(결합)` 블록 연산 시켜 점점 배열 텐서 길이를 키워나가는 O(N) 반복 복잡도 아키텍처입니다. 
def generate(self, idx, max_new_tokens):
    for _ in range(max_new_tokens):  
        
        # 1. Truncation: Sequence(T) 배열 길이가 설정된 설계 스펙 한계 메모리 라인(512)을 넘어가면 배열 뒷부분만 슬라이싱.
        idx_cond = idx[:, -config.block_size:] # 제한 [B, 512] 유지
        
        # 2. 신경망 Forward Propagation 모델 호출. 타겟 데이터가 없으니 loss는 버립니다.
        logits, _ = self(idx_cond)             # [B, T, V_size] 전체 스코어 반환.
        
        # 3. 다음 단어 예측엔 오직 시퀀스의 가장 꼬리(마지막 인덱스인 -1) 로짓만 참조.
        logits = logits[:, -1, :]                # [B, 1차원 구조 축소 차원 V_size] 
        
        # 4. 정규 확률 변환.
        probs = F.softmax(logits, dim=-1)        # [B, V_size]
        
        # 5. 확률 가중치 기반 다항 제비뽑기 시행 (원소 1개 추출). 
        idx_next = torch.multinomial(probs, num_samples=1) # [B, 1]
        
        # 6. 원본 토큰 행렬에 뽑아낸 인덱스 1개를 시간 축 방향(오른쪽)으로 이어 붙입니다.
        idx = torch.cat((idx, idx_next), dim=1)  
        
    return idx
