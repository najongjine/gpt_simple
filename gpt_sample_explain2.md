# 직접 만드는 미니 GPT: 훈련부터 생성까지 (gpt_simple.py)

본 교재는 `gpt_simple.py` 코드를 기반으로 트랜스포머 기반 언어 모델의 핵심 동작 원리를 다룹니다. 특히 모델을 **'학습(Training)할 때'**와 **'생성(Inference/Generation)할 때'** 내부 데이터의 흐름과 텐서의 크기(Shape)가 어떻게 변화하는지 명확하게 비교하며 이해하는 것을 목표로 합니다.

## 1장. GPT 모델의 뼈대와 작동 원리 (준비 단계)
이 장에서는 가장 기초적인 GPT가 어떻게 구성되어 있는지, 그리고 사람이 쓰는 글자를 컴퓨터가 어떻게 이해하는지 아주 쉽게 알아볼 것입니다. 컴퓨터를 전혀 모르는 유치원생도 이해할 수 있도록 재미있는 비유로 시작해, 실제 코드가 어떻게 돌아가는지 공학적인 관점까지 연결해 보겠습니다.

### 1.1 실습 코드(`gpt_simple.py`) 조감도: 모델의 설계도 `GPTConfig`

언어 모델은 기본적으로 **"이전에 나온 단어들을 보고, 그 다음 번에 등장할 가장 자연스러운 단어를 예측하는 기계"**입니다.

이렇게 거대한 기계를 만들기 전에, 우리는 기계의 크기와 성능을 결정하는 설계도가 필요합니다. `gpt_simple.py`에서는 이 설계도를 `GPTConfig`라는 상자(클래스) 안에 모아두었습니다.

```python
class GPTConfig:
    block_size = 512
    vocab_size = 100277 # [중요] cl100k_base 토크나이저 크기
    n_layer = 12
    n_head = 12
    n_embd = 768
    dropout = 0.1
    batch_size = 8
    device = 'cuda' if torch.cuda.is_available() else 'cpu'

config = GPTConfig()
```

#### 🐣 유치원생도 이해하는 비유 설명
우리가 똑똑한 로봇 조수를 만든다고 상상해 보세요!
*   `block_size` (512): **로봇의 단기 기억력**입니다. 로봇이 한 번에 기억할 수 있는 글자가 512개라는 뜻이에요. 이보다 긴 책을 주면 앞부분은 까먹습니다.
*   `vocab_size` (100277): **로봇이 알고 있는 단어책(사전)의 크기**입니다. 이 로봇은 10만 개가 넘는 단어를 알고 있네요!
*   `n_layer` (12) & `n_head` (12): **로봇의 두뇌 층과 눈의 개수**입니다. 12층 탑처럼 여러 번 깊게 생각하고, 12개의 눈으로 문장을 여러 각도에서 동시에 뜯어봅니다.
*   `n_embd` (768): **단어 하나를 이해하기 위한 생각 회선의 수**입니다. 단어 하나를 768개의 특징으로 쪼개서 아주 복잡하게 입체적으로 이해합니다.

#### ⚙️ 컴퓨터 공학 관점의 설명
*   **하이퍼파라미터(Hyperparameter):** 개발자가 직접 설정해 주어야 하는 모델의 뼈대 값들입니다. GPT-2나 GPT-3 같은 거대 모델들은 이 숫자(`n_layer`, `n_embd`)들이 훨씬 더 큽니다.
*   `device`: 그래픽 카드(`cuda`, GPU)를 쓸지, 일반 CPU를 쓸지 결정합니다. 딥러닝은 행렬 곱셈이 무수히 일어나기 때문에 수천 개의 작은 계산기가 동시에 일하는 GPU를 쓰는 것이 압도적으로 빠릅니다.

### 1.2 텍스트와 숫자의 다리 놔주기: 토크나이저와 임베딩

컴퓨터는 '사과', 'Hello' 같은 글자를 전혀 모릅니다. 컴퓨터가 아는 것은 오직 `0`과 `1`로 이루어진 숫자뿐입니다. 그래서 **사람의 글자를 컴퓨터가 계산할 수 있는 숫자로 바꿔주는 과정**이 반드시 필요합니다.

```python
# 토크나이저 설정 및 인코딩 예시
enc = tiktoken.get_encoding("cl100k_base") 
input_text = "Hello, I am a robot."
tokens = enc.encode(input_text)
# 출력 예시: [9906, 11, 314, 1092, 257, 13745, 13]

# Embedding 관련 모델 구조 (GPT 클래스 내부)
self.token_embedding_table = nn.Embedding(config.vocab_size, config.n_embd)
self.position_embedding_table = nn.Embedding(config.block_size, config.n_embd) # 위치 정보
```

#### 🐣 유치원생도 이해하는 비유 설명
**1) 토크나이저(Tokenizer): 글자 조각내기 명인**
*   로봇은 통째로 된 문장을 소화하지 못해요. 그래서 "Hello, I am a robot."이라는 문장을 먹기 좋게 싹둑싹둑 자르는 요리사가 필요합니다. 이 요리사가 바로 **토크나이저**(`tiktoken`)입니다.
*   요리사는 조각낸 단어에 번호표를 붙여줍니다. (예: "Hello" = 9906번, "," = 11번, " I" = 314번). 로봇은 번호표만 보고 "아하, 9906번 단어!" 하고 식별합니다.

**2) 임베딩(Embedding): 숫자표를 풍부한 느낌표로 바꾸기**
*   그런데 9906, 11 같은 번호표는 뜻이 없습니다. 그냥 순번일 뿐이죠. (사과가 1번, 바나나가 2번이라고 사과와 바나나가 1만큼 비슷한 게 아닌 것처럼요).
*   그래서 로봇 두뇌 안쪽(`Embedding`)에서는 이 단순한 번호표를 **'성격 프로필(사전)'**로 바꿉니다. "9906번 단어는 기쁨 수치 80, 인사 수치 95, 사람 연관성 20..." 이런 식으로 768(`n_embd`)개의 점수로 길게 풀어냅니다. 이제야 로봇이 단어의 '느낌(의미)'을 알게 됩니다!

**3) 위치 임베딩(Position Embedding): 순서표 달아주기**
*   "로봇이 사람을 물었다"와 "사람이 로봇을 물었다"는 똑같은 단어들이지만 뜻은 완전히 다릅니다!
*   그래서 단어의 느낌표에다가 **"이 단어는 1번째 자리야", "이 단어는 2번째 자리야"**라는 위치표(Position)를 몰래 하나 더 붙여줍니다. 그래야 로봇이 순서를 헷갈리지 않습니다.

#### ⚙️ 컴퓨터 공학 관점의 설명
*   `tiktoken`: OpenAI가 만든 초고속 BPE(Byte-Pair Encoding) 토크나이저입니다. 자주 같이 쓰이는 글자 묶음을 하나의 토큰(Token)으로 압축합니다. `cl100k_base` 체계를 쓰면 사전에 무려 100,277개의 토큰 종류가 존재하게 됩니다.
*   `nn.Embedding`: Pytorch에서 제공하는 **룩업 테이블(Look-up Table)**입니다. 입력으로 정수 ID가 들어오면, 그 정수에 해당하는 행 번호를 찾아가 `n_embd` 길이의 실수형 벡터(Vector)를 뽑아주는 거대한 엑셀 표와 같습니다. 학습을 거듭할수록 이 엑셀 표 안의 숫자들(가중치)이 점점 단어의 의미를 더 잘 담도록 미세하게 조절됩니다. 
*   단어의 의미적 임베딩(Token)에 순서 정보 텐서(Position)를 단순히 `+` (더하기) 연산으로 합치는 것만으로도 모델은 순서 정보를 완벽하게 학습해낼 수 있습니다!

## 2장. 모델의 심장: 어텐션(Attention) 기법 이해하기
GPT 모델이 그토록 똑똑하게 문맥을 파악할 수 있는 비밀은 바로 **'어텐션(Attention)'**이라는 매커니즘 덕분입니다. 어텐션은 한마디로 **"수많은 단어 중에서 지금 내가 어떤 단어에 가장 집중해야 하는가?"**를 계산하는 과정입니다.

### 2.1 셀프 어텐션(Self-Attention)의 작동 원리 (Q, K, V)

단어들이 서로 대화를 나누며 문맥을 파악하는 공간이 바로 `Head` 클래스입니다.

```python
class Head(nn.Module):
    def __init__(self, head_size):
        super().__init__()
        # 문맥을 파악하기 위한 3가지 질문 도구 생성
        self.key = nn.Linear(config.n_embd, head_size, bias=False)
        self.query = nn.Linear(config.n_embd, head_size, bias=False)
        self.value = nn.Linear(config.n_embd, head_size, bias=False)
        # 하삼각 행렬 (마스킹용)
        self.register_buffer('tril', torch.tril(torch.ones(config.block_size, config.block_size)))

    def forward(self, x):
        # 🟢 입력 데이터 x의 원래 형태: 
        # B(Batch): 동시에 처리할 문장 개수 (예: 8)
        # T(Time): 한 문장 안의 글자(토큰) 개수 (예: 문장 길이)
        # C(Channel/n_embd): 토큰 1개가 가진 특징 변수 개수 (예: 768)
        B, T, C = x.shape  # Shape: [B, T, C]

        # 🟢 1. 토큰들에게 명찰(Key)과 질문(Query) 나눠주기
        # nn.Linear 행렬 곱셈을 통해 차원이 (C -> head_size)로 축소됨
        k = self.key(x)   # Shape: [B, T, head_size] 
        q = self.query(x) # Shape: [B, T, head_size] 
        
        # 🟢 2. 질문과 명찰을 대조하여 친밀도 점수 구하기
        # 연산: q (B, T, head_size) @ k.transpose (B, head_size, T)
        # 모든 단어(T)가 다른 모든 단어(T)들과 1:1로 점수를 매기므로 T x T 행렬이 됨
        wei = q @ k.transpose(-2, -1) * C**-0.5 
        # Shape: [B, T, T] 
        # (예: wei[0][2][1]은 2번째 단어가 1번째 단어에게 주는 점수)
        
        # 🟢 3. [핵심] 미래 단어 컨닝 방지 마스킹
        # 연산: 하삼각행렬의 0인 부분(미래 단어 위치)을 찾아 -inf 점수로 바꿔치기
        wei = wei.masked_fill(self.tril[:T, :T] == 0, float('-inf')) 
        # Shape: [B, T, T] (크기 변화 없음, 값만 -inf로 바뀜)
        
        # 🟢 4. 점수를 확률(0~1)로 예쁘게 포장
        # 연산: Softmax를 거치며 전체 합이 100%가 되도록 변환 (-inf는 0%가 됨)
        wei = F.softmax(wei, dim=-1)
        # Shape: [B, T, T] (크기 변화 없음, 값만 0~1 사이로 바뀜)
        
        # 🟢 5. 실제 알맹이 내용(Value) 추출하기
        # 연산: 차원이 (C -> head_size)로 축소
        v = self.value(x) 
        # Shape: [B, T, head_size]
        
        # 🟢 6. 친밀도 비율(wei)만큼 Value를 섞어서 시너지 내기
        # 연산: wei (B, T, T) @ v (B, T, head_size)
        out = wei @ v 
        # 최종 Output Shape: [B, T, head_size]
        
        return out
```

#### 🐣 유치원생도 이해하는 비유 설명 (Q, K, V)
도서관에서 책을 찾는 상황을 상상해 보세요!
*   **Query (Q - 질문):** 내가 도서관 사서에게 묻는 질문입니다. *"로봇과 관련된 책 있어?"*
*   **Key (K - 명찰/키워드):** 도서관에 꽂혀 있는 책들의 제목 밑에 달린 핵심 키워드 택입니다. *"삐리삐리 인공지능", "맛있는 사과 빙수"*
*   **Value (V - 실제 내용):** 그 책장 안에 들어 있는 진짜 유용한 알맹이 내용입니다.

단어들은 각자 Q, K, V를 하나씩 들고 다닙니다. "사과"라는 단어가 Q(질문)를 던져서 다른 단어들의 K(명찰)들을 살펴봅니다. 어? "먹다"라는 단어의 K(명찰)와 아주 찰떡궁합이네요! 그러면 "사과"는 "먹다"가 가진 V(실제 의미)를 쫙 빨아들여서 자신의 의미를 더욱 풍부하게 업데이트합니다. 

#### ⚙️ 컴퓨터 공학 관점의 설명
*   수식 `q @ k.transpose(-2, -1)`은 Q 벡터와 K 벡터의 **내적(Dot Product)**을 구하는 행렬 곱셈입니다. 두 벡터가 가리키는 방향이 비슷할수록 내적 값(유사도 점수)이 커집니다.
*   `C**-0.5`로 나누어주는 이유(Scaled Dot-Product): 벡터의 차원이 커지면 내적 값이 너무 뻥튀기되어, Softmax 함수 적용 시 기울기(Gradient)가 사라지는 현상을 막기 위한 스케일링 작업입니다.

### 2.2 마스킹(Masking): 왜 미래의 정보를 가려야 하는가? (핵심)

어텐션 연산 중 가장 중요한 코드는 바로 이 한 줄입니다.
`wei = wei.masked_fill(self.tril[:T, :T] == 0, float('-inf'))`

#### 🐣 유치원생도 이해하는 비유 설명
우리가 퀴즈를 풉니다. "나는 매일 아침 밥을 (  ?  )"
빈칸에 들어갈 말을 맞히려면 앞의 "나는", "매일", "아침", "밥을" 이라는 단어만 보고 힌트를 얻어야 합니다.
만약 빈칸 뒤에 원래 정답인 "먹는다"가 스포일러처럼 쓰여 있고 그걸 볼 수 있다면? 퀴즈를 푸는(단어를 예측하는) 연습이 전혀 되지 않겠죠! 

그래서 GPT는 **자신보다 뒤에 나오는(미래의) 단어들은 투명 망토를 씌워서 절대 못 보게** 가려버립니다. 오직 자신과 과거의 단어들끼리만 책갈피(Attention)를 나눌 수 있습니다.

#### ⚙️ 컴퓨터 공학 관점의 설명
`torch.tril()` 함수는 대각선을 기준으로 아래쪽만 1이고 위쪽은 0인 **하삼각 행렬(Lower Triangular Matrix)**을 만듭니다. `wei` 배열(Shape: `B, T, T`)에 이 하삼각 행렬을 뒤집어 씌웁니다.
```text
[1, 0, 0]  <- 1번째 단어는 자기 자신(1번째)만 볼 수 있음
[1, 1, 0]  <- 2번째 단어는 1, 2번째 단어만 볼 수 있음
[1, 1, 1]  <- 3번째 단어는 1, 2, 3번째 단어 모두 볼 수 있음
```
여기서 0인 부분(미래 단어)을 `float('-inf')` (마이너스 무한대)로 바꿔버립니다. 그러면 이어진 `softmax` 함수를 통과할 때 마이너스 무한대는 확률이 `0%`가 되기 때문에, 미래 단어의 Value 정보는 행렬 곱연산(`wei @ v`) 시 완벽하게 차단되어 섞이지 않습니다!

### 2.3 Block과 GPT 구조 완성하기

하나의 Head만 있으면 한 가지 관점에서만 문맥을 파악합니다. 이것을 극복하기 위해 `MultiHeadAttention`과 `Block`이 등장합니다.

```python
class Block(nn.Module):
    def __init__(self, n_embd, n_head):
        super().__init__()
        # 1. 12개의 눈(Head)을 모아 멀티 헤드 어텐션 구성
        head_size = n_embd // n_head
        self.sa = MultiHeadAttention(n_head, head_size)
        
        # 2. 파악한 의미를 깊게 고민하는 생각용 신경망
        self.ffwd = FeedForward(n_embd)
        
        # 3. 데이터가 안정적으로 흐르게 하는 정규화(LayerNorm)
        self.ln1 = nn.LayerNorm(n_embd)
        self.ln2 = nn.LayerNorm(n_embd)

    def forward(self, x):
        # 🟢 입력 차원: [B, T, 가로폭(n_embd)]
        # 🟢 주의: self.sa와 self.ffwd 내부를 통과하더라도 가로폭은 n_embd 그대로 유지됨
        
        # x = 원래 지식(x) + 어텐션으로 파악한 주변 단어 뉘앙스(sa)
        x = x + self.sa(self.ln1(x)) # Shape 변화 없이 유지됨: [B, T, n_embd]
        
        # x = 직전까지의 지식(x) + 깊게 고민해서 결론 내린 내용(ffwd)
        x = x + self.ffwd(self.ln2(x)) # Shape 변화 없이 유지됨: [B, T, n_embd]
        
        return x # 최종 반환되는 크기 역시 [B, T, n_embd]
```

#### 🐣 유치원생도 이해하는 비유 설명
*   **MultiHeadAttention:** 어떤 친구는 단어의 '감정'에만 집중해서 문장을 읽고, 어떤 친구는 단어의 '문법'에 집중해서 읽습니다. 12명(`n_head`)의 친구가 각자 다른 관점에서 문장을 읽고(Self-Attention) 회의를 한 다음, 깨달은 점을 종합하면 문장의 진짜 속뜻을 훨씬 입체적으로 알 수 있겠죠!
*   **FeedForward:** 친구들의 회의가 끝난 후, 각자 방비에 들어가서 "아~ 이 단어는 이런 뜻이겠구나" 하고 한 번 더 곰곰이 생각하며 결론을 소화하는 과정입니다.
*   **Residual Connection (더하기 `x = x + ...`)**: 예전에 알던 원래 뜻을 까먹지 않으면서 거기에 새롭게 깨달은 지식만 살포시 얹어주는 안전장치(잔차 연결)입니다.

## 3장. [상황 A] 모델을 훈련할 때 (Training Phase)
우리가 만든 아기 로봇 GPT가 처음으로 한글(단어)을 떼고 문장을 배우는 훈련 시간에 들어왔습니다! 훈련의 핵심은 **"정답을 미리 쥐여주고, 네가 예측한 것과 정답이 얼마나 틀렸는지(Loss) 혼내는 과정"**입니다. 

### 3.1 훈련용 입력 데이터와 정답(Target) 세팅하기
 GPT가 '기차는 길어 길면 바나나'라는 문장을 배운다고 해볼까요?
*   **입력(Input):** `[기차는, 길어, 길면]`
*   **정답(Target):** `[길어, 길면, 바나나]`
이처럼 정답(Target)은 입력 데이터에서 딱 한 칸씩 오른쪽으로 밀려 있는 미래의 단어들입니다.

### 3.2 훈련 시점의 `forward` 동작: 한 번에, 병렬로 연산하기 (핵심)
아래 코드는 GPT 클래스의 심장부인 `forward` 함수 중, 훈련 시점에 해당하는 부분입니다. 

```python
# GPT 클래스 내부
def forward(self, idx, targets=None):
    # 🟢 1. 훈련용 입력 데이터 (idx) 들어옴
    # B(Batch): 동시에 학습할 문장 묶음 개수 (예: 8)
    # T(Time): 한 문장의 길이 (예: 512 크기 꽉 채움)
    B, T = idx.shape  # Shape: [B, T] (아직 정수형 토큰 ID 상태)

    # 🟢 2. 단어의 의미(tok_emb)와 시작/끝 순서(pos_emb) 부여
    tok_emb = self.token_embedding_table(idx) # Shape: [B, T, C]
    pos_emb = self.position_embedding_table(torch.arange(T, device=config.device)) # Shape: [T, C]
    
    # 🟢 3. 두 의미를 더해서 진짜 입력값(x) 완성
    x = tok_emb + pos_emb # Shape: [B, T, C] (브로드캐스팅으로 차원 자동 적응)
    
    # 🟢 4. 문맥 파악의 방 (12개의 Block) 통과
    # 2장에서 배운 어텐션(Attention) 연산이 여기서 수만 번 일어남!
    x = self.blocks(x)    # Shape: [B, T, C] (크기 변화 없음)
    x = self.ln_f(x)      # Shape: [B, T, C] (정규화)

    # 🟢 5. 최종 단어 예측 점수(Logits) 뽑아내기
    # n_embd(C) 차원을 다시 10만 개의 사전 크기(vocab_size)로 쭉 늘리기
    logits = self.lm_head(x) 
    # Shape 변경: [B, T, C] -> [B, T, vocab_size]
    # (예: "1번째 단어 다음엔 '사과'일 확률 점수 10점, '바나나'일 점수 8점...")
```

#### 🐣 유치원생도 이해하는 비유 설명 (병렬 처리)
*   **입시 학원 쪽지 시험:** 선생님(개발자)이 512칸짜리 빈칸 채우기 시험지를 로봇에게 줍니다. 
*   만약 로봇이 1번 문제 풀고 정답 맞히고, 2번 문제 풀고 정답 맞히는 식으로 한 문제씩 풀면(For 루프 방식) 너무 오래 걸리겠죠?
*   그래서 로봇의 몸통 안에는 수천 명의 꼬마 로봇(GPU)들이 대기하고 있습니다! **"야, 넌 1번 문제 풀어, 난 2번 문제 풀게!" 하면서 512문제를 한 방에 동시에 풉니다** (병렬 처리).
*   **이게 가능한 이유:** 2장에서 배운 **마스킹(Masking)** 덕분입니다! 512문제를 동시에 책상에 깔아놔도 마스킹(투명 망토) 때문에 뒷문제의 정답을 절대 훔쳐볼 수 없기 때문이죠!

---

### 3.3 정답과 비교하기: 손실(Loss) 점수 계산

모델이 열심히 정답을 찍어서 냈다면(Logits), 이제 진짜 정답(`targets`)과 비교해 얼마나 틀렸는지 오차(Loss) 점수를 매겨야 합니다.

```python
    loss = None
    # 🟢 6. 훈련 상황: 선생님이 정답지(targets)를 줬을 때
    if targets is not None:
        B, T, C = logits.shape # 현재 C는 vocab_size (100,277)
        
        # 🟢 7. [핵심] 차원 무너뜨리기 (평탄화, Flatten)
        # B(문장 개수) 장의 T(단어 개수) 문제지를 한 줄로 길게 이어붙임
        logits = logits.view(B*T, C)      # Shape: [B*T, vocab_size]
        targets = targets.view(B*T)       # Shape: [B*T]
        
        # 🟢 8. 정답과 비교하여 오차(Loss) 계산
        # 10만 개 단어 중 정답 단어에 얼마나 높은 점수를 줬는지 크로스 엔트로피로 채점
        loss = F.cross_entropy(logits, targets) 

    # 최종 반환 (훈련 시 loss가 반드시 존재함)
    return logits, loss
```

#### ⚙️ 컴퓨터 공학 관점의 설명 (Loss Calculation & View)
*   `view(B*T, C)` 동작 원리: 파이토치(PyTorch)의 손실 함수인 `F.cross_entropy`는 기본적으로 2차원 텐서 `[N, Class_Count]` 형태를 입력으로 기대합니다.
*   그래서 원래 `[배치 크기(B), 문장 길이(T), 단어 사전 크기(vocab_size)]`였던 3차원 바둑판 모양의 예측값을 쭈욱 펴서 **`[(B × T)개의 토큰, 단어 사전 크기]` 형태의 2차원 행렬로 납작하게 만들어 줍니다.** 
*   마찬가지로 정답지(`targets`)도 `[B, T]` 형태에서 `[B × T]`의 1차원 배열로 길게 늘어뜨려 줄을 세워 점수를 매깁니다. 이 Loss 값이 계산되어야 비로소 `loss.backward()`를 통해 모델이 학습(가중치 업데이트)될 수 있습니다!

## 4장. [상황 B] 모델을 사용할 때 (Inference / Generation Phase)
혹독한 훈련을 마치고 드디어 실전에 투입된 똑똑한 GPT입니다. 이제 정답지는 더 이상 없습니다. 사용자가 채팅창에 입력한 글자(프롬프트) 하나만 가지고 순수하게 자신의 실력으로 뒷말을 한 글자씩 지어내야 합니다.

### 4.1 새로운 단어 생성의 기본: `generate` 함수 해부
실전 상황은 `generate` 함수 내부에서 일어납니다. 이 함수는 방금 배운 `forward` 함수를 수십 번, 수백 번 반복해서 부르면서 단어를 한 칸씩 덧붙여 나가는 공장 레일과 같습니다.

```python
# GPT 클래스 내부
# idx: 사용자가 처음 입력한 프롬프트 토큰들
# max_new_tokens: 지어낼 새로운 글자(토큰) 개수 (예: 20글자 쓰기)
def generate(self, idx, max_new_tokens):
    # 🔴 1. 글자를 계속 이어붙이기 위해 지정한 횟수만큼 "반복(For 루프)"
    for _ in range(max_new_tokens):
        
        # 🔴 2. [핵심] 모델의 기억 한계치(block_size)를 유지하기 위해 앞부분 잘라내기
        # 토큰 수가 512개를 넘어가면 제일 옛날 기억은 싹둑 잘라서 버립니다.
        idx_cond = idx[:, -config.block_size:] # Shape: [B, T<=512]
        
        # 🔴 3. 모델(forward)에 집어넣어 다음에 올 후보 단어 점수 뽑기!
        # 실전이므로 정답(targets)이 주어지지 않음 -> 내부 계산에서 loss는 무조건 None이 됨!
        logits, _ = self(idx_cond) 
        
        # 🔴 4. "가장 최근(마지막)에 읽은 단어"의 예측 점수만 쏙 빼오기
        logits = logits[:, -1, :] # Shape: [B, 가로로 쫙 깔린 10만 단어 점수표]
        
        # 🔴 5. 점수표를 "동전 던지기용 확률 비율(0~100%)"로 예쁘게 가공 
        probs = F.softmax(logits, dim=-1)
        
        # 🔴 6. 확률에 입각해 주사위를 굴려서(샘플링) 다음 단어 1개 뽑기!
        # (예: 사과 90%, 바나나 10%면 웬만해선 사과가 뽑힘)
        idx_next = torch.multinomial(probs, num_samples=1) # Shape: [B, 1]
        
        # 🔴 7. 방금 뽑은 단어를 기존 문장 맨 뒤에 테이프로 철썩 이어붙임
        idx = torch.cat((idx, idx_next), dim=1) # Shape 길이 T가 1만큼 길어짐
        
    return idx # 20번 다 돌았으면 길어진 최종 문장을 돌려줌
```

#### 🐣 유치원생도 이해하는 비유 설명 (자동 회귀, Auto-Regressive)
*   사용자가 GPT 로봇에게 빈 종이에 **"나는"** 이라고 적어서 줬습니다 (`idx_cond`).
*   **첫 번째 루프:** 로봇이 종이를 쓱 보더니 **"밥을"**이라는 카드를 하나 뽑습니다 (`idx_next`). 그리고는 종이에 **"나는 밥을"**이라고 붙입니다 (`torch.cat`).
*   **두 번째 루프:** 카드가 합쳐진 종이를 그대로 로봇에게 *다시* 가져다줍니다! (왜 `For 루프`를 도는지 아시겠죠?) 이번엔 종이를 다시 처음부터 읽다가 **"먹는다"**를 뽑습니다. 종이는 **"나는 밥을 먹는다"** 가 됩니다.
*   이렇게 `max_new_tokens=20`이라면? 위 행동을 무식하게 20번 뺑뺑이 도는 겁니다! 이렇게 자기가 쓴 글을 다시 읽고 다음 걸 쓰고, 또 다 읽고 다음 걸 쓰는 걸 **"자동 회귀(Auto-Regressive)"**라고 부릅니다.

#### ⚙️ 컴퓨터 공학 관점의 설명 (`idx_cond` 자르기)
*   `idx[:, -config.block_size:]`: GPT의 단기 기억력 한계치는 `GPTConfig`에서 정한 `512`개입니다. 하지만 루프를 돌면서 단어를 계속 이어붙이면(`torch.cat`) 배열의 길이 `T`가 513, 514... 무제한으로 길어집니다!
*   그래서 `idx`를 모델에 넣기 직전에 리스트 슬라이싱(`[-512:]`)을 통해 뒷부분에서부터 딱 `block_size`만큼만 잘라와서 모델에 안전하게 통과시키는 테크닉입니다.

## 5장. 총정리 포인트: 훈련(Training) vs 사용(Generation) 전격 비교

그렇다면 가장 헷갈리는 훈련 때의 동작(`forward`)과 실제 사용할 때의 동작(`generate`)의 차이를 한눈에 요약해 봅시다.

| 구분 | 📝 훈련할 때 (Training) | 🤖 사용할 때 (Generation) |
| :--- | :--- | :--- |
| **비유** | 학원에서 빈칸 512개짜리 쪽지 시험치고 혼나는 중 | 실전에서 자기가 적은 글을 다시 읽으며 글쓰기 대회 하는 중 |
| **작동 위치** | `forward(idx, targets=정답)` | `generate(idx)` $\rightarrow$ 내부에서 수십 번 `forward` 호출! |
| **목적** | 10만 단어 중 정답을 제대로 찍었는지 확인하고 **오차(Loss) 줄이기** | 확률에 맞게 가장 자연스러운 **다음 단어를 예측(추론)하기** |
| **선생님(Target) 유무** | 있음! (한 칸씩 밀려 있는 진짜 정답 시퀀스) | **없음!** (오직 입력 프롬프트(`idx`)만 모델에 들어감) |
| **손실(Loss) 점수** | `F.cross_entropy`로 빡세게 계산되어 채점됨 | `loss = None` (정답지가 없으므로 **채점 불가능, 당연히 None**) |
| **처리 방식 (속도)** | 병렬 처리 고속도로 (For문 없음, 행렬로 **512 단어를 한꺼번에 연산**) | 달팽이 처리 (글자 수만큼 **For 루프를 돌며 한 칸씩 한 칸씩 덧붙임**) |

### 5.1 텐서 모양(Shape)이 변하는 드라마틱한 차이 비교표

`"나는 밥을"` 이라는 두 글자($T=2$)에서 시작한다고 가정합시다.

#### 📝 [훈련(Training) 시 텐서 흐름]
> 훈련 때는 정답지(Target)가 있기 때문에 단어가 눈덩이처럼 달라붙지 않고 고정된 모양으로 한 방에 쑥 통과합니다.

*   `idx`: `[배치: 8, 원본 길이: 2]` 
*   `targets`: `[배치: 8, 정답 길이: 2]`
*   `tok_emb` 통과: `[8, 2, 임베딩폭: 768]` 
*   어텐션 통과: `[8, 2, 768]`
*   **Loss 계산**: `logits.view(8*2, 100277)` 쭈욱 펴서 $16$문제 한 방에 채점!

#### 🤖 [사용/추론(Inference) 시 텐서 흐름 (For 루프)]
> 루프를 한 번 돌 때마다 모델을 완전히 처음부터 빠져나오며 $T$가 계속 오른쪽으로 자라납니다.

| 루프(Loop) 횟수 | 입력 텐서(`idx`) Shape | 출력 텐서(`idx_next`) Shape | 이어붙인 결과 (`torch.cat`) Shape |
| :--- | :--- | :--- | :--- |
| **시작 전 (입력)** | `[B: 8, T: 2]` ("나는 밥을") | - | - |
| **1번째 루프** | `[8, 2]` | `[8, 1]` ("먹는다") 뽑음 | `[8, 2 + 1 = 3]` ("나는 밥을 먹는다") |
| **2번째 루프** | `[8, 3]` | `[8, 1]` ("맛있게") 뽑음 | `[8, 3 + 1 = 4]` ("나는 밥을 먹는다 맛있게") |
| **3번째 루프** | `[8, 4]` | `[8, 1]` ("!") 뽑음 | `[8, 4 + 1 = 5]` ("나는 밥을 먹는다 맛있게 !") |
| ... | ... | ... | ... |

*   **진짜 마지막 참고:** 훈련할 때나 생성할 때나 **"하삼각 확률 마스킹(`tril`, 컨닝 방지용)"은 무조건, 언제나, 항상 적용**됩니다! (생성할 때도 자기가 쓴 예전 글을 읽을 때는 과거만 읽어야지 미래를 볼 수 없기 때문입니다.)
