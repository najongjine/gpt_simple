🏗️ 2. 공장 설계도 그리기 (GPTConfig) 

class GPTConfig:
    block_size = 512    # 기억할 수 있는 문장의 길이
    vocab_size = 100277 # 알고 있는 단어장의 크기
    n_layer = 12        # 공장의 층 수
    n_head = 12         # 소통방(회의실)의 개수
    n_embd = 768        # 단어의 복잡도
    dropout = 0.1
    batch_size = 8
    device = 'cuda' if torch.cuda.is_available() else 'cpu'

💻 컴공 교수님 설명

트랜스포머는 RNN과 달리 단어를 순차적으로 처리하지 않고 한 번에 통째로 처리합니다. 따라서 단어의 순서(Sequence) 정보를 알 수 없습니다.

이를 해결하기 위해 각 토큰의 의미를 벡터화하는 `Token Embedding`과 토큰의 절대적인 위치값을 벡터화하는 `Positional Embedding`을 생성한 뒤, 이 두 텐서를 덧셈(Element-wise Add)형태로 합쳐서 모델의 병렬 처리가 가능하도록 만듭니다.

🗣️ 4. 공장의 심장! 단어들끼리 회의하기 (Attention)

🔥 코드 중 가장 중요한 부분이 바로 이 `Head`와 `MultiHeadAttention` 입니다!

class Head.forward 
# 핵심 수학 공식!
k = self.key(x)     # 내가 가진 특징 (Key)
q = self.query(x)   # 내가 찾고있는 특징 (Query)
v = self.value(x)   # 내가 공유할 진짜 정보 (Value)

# 단어끼리 얼마나 친한지 점수계산
wei = q @ k.transpose(-2, -1) * (C**-0.5)

# (중요!) 미래의 단어를 컨닝 못하게 막기
wei = wei.masked_fill(self.tril[:T, :T] == 0, float('-inf'))
wei = F.softmax(wei, dim=-1) # 확률로 바꿈 (합 100%)

# 정보를 점수만큼 섞어버리기
out = wei @ v

🐥 놀이방 선생님 설명

문장: "사과를 샀었어. 내일 그것을 먹을거야."

Q. 여기서 그것은 무엇일까요? 누가 봐도 '사과'죠!

이걸 모델이 어떻게 알아낼까요?
그것(Query): "나는 과일이나 음식이라는 정보를 찾고 있어!"
사과(Key): "앗, 내가 과일이야!"
둘의 성향이 딱 맞으니(q @ k), 매우 높은 연관성 점수(wei)를 줍니다. 그래서 '그것'이라는 단어는 결국 '사과'의 의미를 듬뿍 흡수하게 됩니다! 이것이 바로 어텐션(Attention, 집중)입니다.

💻 컴공 교수님 설명

이것이 바로 Google이 발표한 Attention Is All You Need 논문의 핵심인 Self-Attention 메커니즘입니다.
각 단어 벡터 `x`에 서로 다른 가중치 행렬을 곱하여 `Q, K, V`(Query, Key, Value)를 생성합니다.

  • q @ k.transpose : 두 행렬의 내적(Dot Product)을 구합니다. 벡터가 유사할수록 내적 값이 엄청 커집니다. (유사도 구하기)
  • C**-0.5로 나누기 : 차원수(C)가 커질수록 내적값이 팽창하여 Softmax에서 기울기 소실 문제가 발생하므로, 루트(Scale)로 나누어줍니다. (Scaled Dot-Product)
  • masked_fill : 언어 모델은 "다음에 올 단어를 예측"하는 것이므로, `t` 시점의 단어를 예측할 때 `t+1` 시점의 미래 토큰의 정보를 앞당겨 보면(커닝) 안됩니다! 그래서 삼각행렬(tril)을 이용해 미래 시점의 점수를 -∞(무한대)로 덮어버립니다.
이 하나의 소통방(Head)만 있으면 아쉬우니, 이걸 여러 개(12개)로 나란히 놓고 다양한 관점으로 생각하는 것이 위의 코드에 있는 MultiHeadAttention 클래스입니다.

💭 5. 혼자만의 생각 정리 시간 (FeedForward) 

class FeedForward(nn.Module):
    def __init__(self, n_embd):
        super().__init__()
        self.net = nn.Sequential(
            # 생각을 4배로 확장시킴
            nn.Linear(n_embd, 4 * n_embd),
            # 복잡하게 꼬아주기 (비선형성)
            nn.ReLU(),
            # 다시 원래 크기로 압축
            nn.Linear(4 * n_embd, n_embd),
        )

    def forward(self, x):
        return self.net(x)

🐥 놀이방 선생님 설명

친구들과의 소통방(Attention) 회의가 다 끝났습니다! 다른 단어들이 나에게 무슨 정보를 줬는지 가득 안고 각자 자기 자리로 돌아옵니다.

이제 남의 얘기를 듣는 건 멈추고, "음...이 모든 정보를 어떻게 이해하는게 맞을까?" 혼자 깊이 생각(FeedForward)하며 정보를 정리하는 개인 학습 시간이에요.

💻 컴공 교수님 설명

Attention에서는 문장 내 토큰들 간의 관계(공간적 맥락)를 학습했습니다. 반면 MLP 계층인 FeedForward Layer는 토큰끼리 섞이지 않고, 각 토큰 벡터의 개별적인 특징(차원)을 깊게 비선형 변환하는 역할을 합니다.
차원을 4배로 늘렸다(`4*n_embd`)가 복구하는 병목 구조(Bottleneck)를 차용해 풍부한 특징을 학습시킵니다.

🧱 6. 모든 부품 합치기 (Block) 

class Block(nn.Module):
    # (설정 부분 생략) ...
    def forward(self, x):
        # 1. 소통의 방(Attention)에 다녀온 결과를 기존 x에 더한다!
        x = x + self.sa(self.ln1(x))
        # 2. 개인 생각(FeedForward) 결과를 또 더해준다!
        x = x + self.ffwd(self.ln2(x))
        return x
💡
더하기 기호( `x = x + ...` )는 왜 있을까요?
이걸 잔차 연결(Residual Connection)이라고 부릅니다. 유치원생 버전으로 말하자면 "공장에 새로운 물건만 돌아다니면 원래 내가 누구였는지 까먹으니까, 방금 전의 내 원래 모습(x)을 안전하게 계속 들고 다니자!" 입니다. (컴공 전공자에게는 깊은 신경망을 지날 때 발생하는 기울기 소실(Vanishing Gradient)을 막기 위한 지름길 허브(Skip Connection)로 이해하면 됩니다.)

🚀 7. 공장이 굴러가다! 다음 단어 생성 (Generate)

`GPT` 본체 클래스는 지금까지 만든 1~6 과정의 부품을 조립해놓은 것입니다.
이제 학습된 GPT 모델에게 "다음 단어 내놔봐!"라고 시키는 함수가 바로 generate 입니다. 

def generate(self, idx, max_new_tokens):
    for _ in range(max_new_tokens): # 몇 개의 단어를 만들지 반복!
        # 문장이 너무 길면 모델 기억력(block_size)만큼 잘라냄
        idx_cond = idx[:, -config.block_size:] 
        
        # GPT 공장에 집어넣어서 결과물(추측한 확률표)을 가져옴
        logits, _ = self(idx_cond) 
        
        # 마지막 토큰(금방 들어온 단어)의 예측표만 가져옴
        logits = logits[:, -1, :] 
        
        # 100% 확률이 되도록 보기좋게 바꿈 (Softmax)
        probs = F.softmax(logits, dim=-1) 
        
        # 주사위를 굴려서 가장 그럴싸한 단어 1개 뽑기!
        idx_next = torch.multinomial(probs, num_samples=1) 
        
        # 우리가 적었던 문장 뒤에 방금 뽑은 새 단어를 붙인다!
        idx = torch.cat((idx, idx_next), dim=1) 
        
    return idx # 새 단어가 줄줄이 붙은 최종 문장 반환

💻 컴공 교수님 설명

이 과정이 바로 모델이 문장을 생성하는 핵심인 자기회귀(Auto-Regressive) 프로세스입니다.

결과(새 토큰)가 다시 입력의 꼬리에 달라붙어(Concatenate) 다음 상태를 예측합니다. 이 때 가장 높게 나온 단어만 뽑는 것(`argmax`)이 아니라 `multinomial` 함수를 쓰는 이유는, 모델에 다양성을 부여하여 늘 똑같은 답만 하지 않도록 마치 주사위 굴리듯 확률에 따른 샘플링(Sampling) 연산을 수행하기 위함입니다.
